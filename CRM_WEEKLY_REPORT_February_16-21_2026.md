## What was done, by day

### Monday 16 February
**Notifications and alerts**
- Notifications when you’re not on the app (push) were made more reliable, with better handling of errors and missing data.
- The notification bell now updates the unread count when you return to the tab, so the number is accurate.
- The notification list and badge stay in sync with the server.
- The buttons to turn push notifications on or off were improved for accessibility (e.g. clearer labels for screen readers).
- Setup and configuration for push notifications were updated and old documentation was removed.

---

### Tuesday 17 February
**Dashboard**
- New filter option to see **all** data (no date limit).
- You can filter by **campaign** and by **date range** (e.g. this week, this month, or a custom range).
- The dashboard now separates **follow-up tasks** from other tasks so you can see each type clearly.
- **Charts and stats** use the new filters and show data by campaign and date.
- The dashboard and several other pages (reports, activity logs, user management, etc.) were updated to **load faster**, with clearer loading indicators.

### Thursday 19 February
**Dashboard**
- New **report charts** were added to the main dashboard, with clearer labels so data is easier to read.

**Tasks**
